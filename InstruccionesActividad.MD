## 2. Objetivos del Proyecto
[cite_start]El propósito principal es que el alumnado comprenda que el despliegue técnico tiene un impacto real en la salud, la economía y la seguridad de las personas[cite: 11, 12, 13].

* [cite_start]**Criterios de Evaluación:** * No se evaluará el código ni la programación[cite: 14].
    * [cite_start]Se evaluará la capacidad de comunicación, razonamiento y conocimientos técnicos aplicados[cite: 15].
* [cite_start]**Meta en el aula:** Eliminar la visión abstracta del módulo y que el alumno vea el impacto real de su trabajo[cite: 59, 60].

## 3. Estructura Obligatoria de la Presentación
[cite_start]Cualquier presentación del grupo debe seguir este orden[cite: 17]:
1. [cite_start]**Contexto del sector:** Problema real existente[cite: 18].
2. [cite_start]**Aplicación propuesta:** Solución técnica[cite: 19].
3. [cite_start]**Arquitectura de despliegue:** Explicada para un público no informático[cite: 20].
4. [cite_start]**Importancia del despliegue:** Relevancia en el sector salud[cite: 21].
5. [cite_start]**Riesgos reales:** Consecuencias si el sistema falla[cite: 22].
6. [cite_start]**Mini-demo:** Esquema visual o demostración rápida[cite: 23].
7. [cite_start]**Dinámica interactiva:** Actividad final con los compañeros[cite: 24].

---

## 4. Especialidad: Farmacia (Gestión de Medicamentos y Pacientes)

### Contexto de Trabajo
[cite_start]Las farmacias operan con elementos críticos que requieren alta disponibilidad[cite: 26, 27]:
* [cite_start]Control de stock de medicamentos[cite: 29].
* [cite_start]Procesamiento de recetas electrónicas[cite: 30].
* [cite_start]Manejo de datos sensibles de pacientes[cite: 31].

### Detalles del Software y Despliegue
* [cite_start]**La Aplicación:** Sistema web para gestión de stock y validación de recetas, accesible por farmacéuticos y técnicos[cite: 33, 34, 35, 36].
* **Infraestructura de Despliegue:**
    * [cite_start]Servidor Linux[cite: 38].
    * [cite_start]Contenedores Docker para aislar servicios[cite: 39].
    * [cite_start]Nginx configurado como proxy inverso[cite: 40].
    * [cite_start]Uso obligatorio de protocolo seguro HTTPS[cite: 41].
    * [cite_start]Variables de entorno para la gestión segura de credenciales[cite: 42].

### Conceptos Clave de DAW Aplicados
* [cite_start]Despliegue seguro y alta disponibilidad[cite: 44, 45].
* [cite_start]Estrategias de Backups[cite: 46].
* [cite_start]Diferenciación entre entornos de producción y pruebas[cite: 47].

> [cite_start]**Riesgo Crítico:** Si el despliegue falla, se interrumpe la dispensación de medicamentos a los ciudadanos[cite: 48, 49].
